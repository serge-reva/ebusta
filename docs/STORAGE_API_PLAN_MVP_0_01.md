# STORAGE_API_PLAN_MVP_0_01.md

## План API для хранения (скачивание файлов) – MVP 0.01

### Цель
Обеспечить возможность скачивания файлов книг по SHA1 через HTTP.

### Внешний API (downloader)

GET /books/{sha1}
HEAD /books/{sha1}
GET /books/{sha1}?meta=1
text


- `sha1` – 40-символьный hex-идентификатор.
- Если `?meta=1` – возвращается JSON с метаданными книги.
- HEAD-запрос возвращает заголовки `Content-Length` и `Content-Disposition`.
- При успехе статус `200`, тело – поток байт.
- При ошибке – JSON с полем `error` (см. API_ERROR_MAPPING.md).

### Внутренний gRPC-интерфейс (между downloader и plasma-node)
Используется стандартный `StorageNode` (см. `downloads.proto`).

### Валидация
- `sha1` проверяется на соответствие формату (40 hex, нижний регистр) – если нет, `400 Bad Request`.
- Размер файла не ограничен на уровне downloader (лимиты накладывает plasma-node).

### TraceID
- Клиент может передать `X-Trace-Id`.
- downloader генерирует, если отсутствует, и прокидывает в gRPC-метаданные к plasma.

### Критерии готовности
- `make e2e-download` проверяет:
  - HEAD существующей книги возвращает 200 и Content-Length.
  - GET существующей книги возвращает 200 и тело соответствует размеру.
  - GET с `?meta=1` возвращает валидный JSON.
  - GET несуществующей книги → 404 с JSON-ошибкой.
  - GET с невалидным SHA1 → 400.
