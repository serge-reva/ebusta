# PROTO_IMMUTABILITY.md

## Неизменяемость Proto-контрактов

### Основные правила
- Не изменять существующие номера полей.
- Не изменять типы полей.
- Не удалять поля напрямую – помечать как `reserved`.
- Новые поля добавлять только с новыми номерами.
- При переименовании сохранять wire-совместимость (не менять номер).

### Версионирование
- Пакет proto имеет версию в имени, например `libraryv1`.
- **Ломающие изменения** разрешены только в новом пакете (`libraryv2`).
- `libraryv1` должен оставаться обратно совместимым.

### Проверки
- `make proto-lint` – проверка стиля и рекомендаций (buf lint).
- `make proto-breaking` – проверка на обратную совместимость с предыдущей версией (buf breaking).
