# RUNBOOK

Version: 1.1  
Last Updated: 2026-03-02

## Host Loop
```bash
make proto
make build
make up
make down
```

## Docker Runtime Loop
```bash
make certs-generate
make docker-build
make docker-up
make docker-status
make docker-down
```

## Internal gRPC mTLS
- Internal gRPC services use mTLS with a shared CA.
- Certs are generated by `scripts/gen-certs.sh` into `certs/`.
- Docker compose mounts certs as read-only: `./certs:/certs:ro`.
- gRPC health checks use TLS and client certificates.

## Quick Checks
```bash
curl -i http://localhost:8443/health
docker compose ps
```

## Negative Check (mTLS enforced)
From a container, plaintext probe to internal gRPC must fail:
```bash
docker compose exec -T datamanager /app/grpc_health_probe -addr=datamanager:50051
```
